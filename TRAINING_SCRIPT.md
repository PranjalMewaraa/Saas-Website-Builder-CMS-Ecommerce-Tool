# Training Script
## Live Onboarding Demo (45–60 min)

## Audience
New merchant/admin users.

## Outcome
By end of session, user can:
- create store and catalog basics,
- edit page visually,
- publish site,
- process an order.

---

## Segment 1 (5 min): Platform Orientation
Say:
- “You have one admin for content and commerce.”
- “Always confirm site and store context first.”

Show:
- Main nav areas: Content, Stores/Catalog, Manage

---

## Segment 2 (10 min): Store + Catalog Setup
Do:
1. Open `Stores`, create/select store.
2. Open `Brands`, add one brand.
3. Open `Categories`, create one category with 2 attributes.
4. Open `Products`, create one product with values.

Highlight:
- Store-scoped model (wrong scope = wrong data view).

---

## Segment 3 (15 min): Visual Page Editing
Do:
1. Open `Content → Pages → Edit`.
2. Add `Layout/Section` block.
3. Add row/col/atomic blocks.
4. Edit text/image/button styles.
5. Configure Hero and Footer examples:
   - Hero basic background color
   - Footer menu groups with titles

Explain:
- Inspector controls selected element only.
- Draft changes are not live until publish.

---

## Segment 4 (8 min): Menus, Forms, SEO
Do:
1. Create/update a menu and assign to header/footer.
2. Add a form block and pick a form.
3. Set page/site SEO basics.

Explain:
- Forms submissions can be reviewed in admin manage area.

---

## Segment 5 (7 min): Publish + Verify
Do:
1. Open `Content → Publish`.
2. Publish snapshot.
3. Open storefront URL.
4. Verify key pages render correctly.

Explain:
- Preview token is for draft mode only.

---

## Segment 6 (10 min): Ops Walkthrough (Order + Inventory)
Do:
1. Place test order on storefront.
2. Open `Manage → Orders`, view details, change status.
3. Open `Manage → Inventory`, adjust stock.

Explain:
- Keep order and inventory review in daily routine.

---

## Q&A Prompts
- “Do you know how to verify store scope before editing?”
- “Can you publish and validate storefront in one pass?”
- “Can you update category attributes and see effect in product forms?”

---

## Hands-On Tasks for Trainee
1. Create one new category with three attributes.
2. Add one product with one variant image.
3. Build one section on home page.
4. Publish and verify storefront.
5. Update one test order status.

---

## Trainer Checklist
- [ ] User understood scope model
- [ ] User can add/edit content blocks
- [ ] User can manage categories/products
- [ ] User can publish snapshot
- [ ] User can process order/inventory action
